# Debugging Guide: Celim_GLPI RPA

This guide provides a comprehensive overview of the `Celim_GLPI` project structure, execution flow, and specific instructions for debugging and troubleshooting.

## 1. Project Structure & Components

The project is an RPA (Robotic Process Automation) built in Python, designed to monitor and alert on GLPI tickets.

### Core Class: `GLPI`
- **`__init__`**: Initializes parameters, sets up Telegram IDs, loads configuration files (`horatermino.txt`, `caminho_excel.txt`, `sistema.txt`), and checks for already notified tickets in the database.
- **`relogio_timer(tempo)`**: A custom countdown timer for the wait interval between loops.
- **`limparDescricao(descricao)`**: Uses regex and HTML unescape to clean ticket content for Telegram messages.
- **`tempoComercial(data_inicio, data_fim)`**: **Critical Logic.** Calculates the business time (excluding weekends and holidays) between two dates. It fetches holidays from the `glpi_holidays` table.
- **`novoChamado()`**: Queries the GLPI database for new tickets (status=1) with an ID greater than the last processed one. Sends alerts via Telegram and logs to `tb015_glpi`.
- **`chamadosVencendoWeb()`**: Uses **Selenium (headless)** to log into the GLPI web interface, navigate to the ticket list, and identify tickets nearing their SLA (TA/TS > 80%).
- **`chamadosPendente()`**: Generates an Excel report of pending tickets, opens it, takes a screenshot using Paint, and sends the image via Telegram.

### External Libraries (`01_lib`)
The script depends on several shared modules located in a central repository (defined in `Config\Diretorios\RaizProjeto.txt`):
- `BotVar`: Global parameters and Telegram bot instance.
- `BotLog`: Logging to files, console, and Excel generation.
- `BotTarefas`: Manages RPA status in the database (e.g., "Executando", "Parado").
- `BotConectarBd`: Handles MySQL/MariaDB queries.
- `BotGerenciadorJanelas`: Window focus and title management for GUI automation.

---

## 2. Execution Flow

1.  **Initialization**: Loads configuration, sets up database connections, and initializes the `GLPI` class.
2.  **Main Loop**: Runs until the `horariotermino` is reached.
    -   **Step A: `chamadosPendente`**: If `enviar_pendentes` is true, generates and sends the Excel screenshot.
    -   **Step B: `novoChamado`**: Checks DB for new tickets and alerts.
    -   **Step C: `chamadosVencendoWeb`**: Scrapes GLPI web for SLA alerts.
    -   **Step D: `relogio_timer`**: Waits for the configured interval (`tempo`).
3.  **Finalization**: Calls `BotFinalizar.finalizarExecucao` to log the end of the process.

## 3. How to Debug

### A. Testing Specific Modules
To debug a specific functionality without running the full loop, comment out the main loop and call the method directly after `bot = GLPI()`:
```python
bot = GLPI()
# bot.novoChamado()         # Test DB ticket detection
# bot.chamadosVencendoWeb() # Test Selenium scraping
# bot.chamadosPendente()    # Test Excel/Paint automation
```

### B. Selenium Debugging
The `chamadosVencendoWeb` method runs in **headless mode**. To see what the bot is doing:
1.  Locate line ~450: `options.add_argument("--headless=new")`.
2.  Comment it out: `# options.add_argument("--headless=new")`.
3.  Ensure `chromedriver.exe` matches your installed Chrome version.

### C. SLA & Business Time Logic
If alerts are firing at wrong times, debug `tempoComercial`:
- Check the `glpi_holidays` table in the GLPI database.
- Print `data_inicio_dt` and `data_fim_dt` inside the loop to track how it's incrementing days.

### D. GUI Automation (Excel/Paint)
The `chamadosPendente` module is sensitive to window focus:
- If it fails, check if `BotGerenciadorJanelas.esperarTitulo` is timing out.
- Ensure the OS language matches the expected titles (e.g., "Save As" vs "Salvar como" logic in lines ~1080).
- **Tip**: Keep the mouse and keyboard untouched during this phase.

### E. Database Issues
- The bot uses two different DB contexts:
    1.  `glpi`: Read-only access to ticket and holiday tables.
    2.  `BotVar.bancomysql`: Write access for RPA tracking (`tb015_glpi`).
- Ensure the user defined in `BotVar` has permissions for both.

## 4. Common Troubleshooting

| Issue | Potential Cause |
| :--- | :--- |
| **ModuleNotFoundError: 'GerarChave'** | `Config\Diretorios\RaizProjeto.txt` points to an incorrect path for `01_lib`. |
| **Selenium Timeout** | GLPI web interface changed, or `chromedriver.exe` is outdated. Check XPaths. |
| **Telegram Not Sending** | Bot token is invalid or the Chat ID in `BotVar` parameters is wrong. |
| **Excel/Paint Error** | `subprocess.Popen` failed to find the applications. Check `Config\caminho_excel.txt`. |
| **Loop Terminates Early** | Check `Config\horatermino.txt`. It must be in `HH:MM` format. |

## 5. Logs Location
Check the directory generated by `BotLog`. It usually follows this pattern:
`log\log[BOT_NAME]_[DATE]\`
- `dataframe\`: Contains Excel exports of queries.
- `print\`: Contains screenshots from the `chamadosPendente` module.
- Console output: Detailed step-by-step trace of every module.
